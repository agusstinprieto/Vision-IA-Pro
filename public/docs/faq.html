Protocolos de Automatización, Fallas de Hardware y Matriz de Escalación.

1. Fallas de Hardware (Cámaras/Sensores)

¿Qué pasa si una cámara se daña?

El sistema usa un monitor de "Latidos" (Heartbeat). Si el Servidor Local no recibe respuesta de una cámara por 1.5 minutos (3 pings fallidos), la marca automátiamente como OFFLINE.

Acción Automática:

Generación de evento DEVICE_HEALTH_ALERT -> Envío a Supabase -> Notificación Dashboard.

¿Cómo se entera soporte técnico?

No se requiere reporte humano. Supabase dispara un Webhook directo a su sistema de tickets (Jira/Zendesk) o envía un correo urgente a soporte@ia-agus.com.

2. Automatización & Mantenimiento

  • Mantenimiento Preventivo

    El sistema cuenta horas de operación y sugiere limpieza de lentes automáticamente cada 8,000 horas.

  • Cortes de Energía

    El servidor (con UPS) envía una señal de "Power Loss" antes de apagarse de forma segura para evitar corrupción de datos.

  • Internet Lento

    Priorización de tráfico (QoS). Las Alertas Críticas (JSON ligero) salen primero. El video evidencia se pausa hasta recuperar ancho de banda.
